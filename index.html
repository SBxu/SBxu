CSRF

Cross-site request forgery 跨站请求伪造
是一种挟制用户在当前已登录的Web应用程序上执行非本意的操作的攻击方法。

cross-site request 跨站请求 是受害者执行的
forgery 伪造 是由攻击者操作的

  • 受害者角度
    用户在当前已经登录的Web应用程序上执行非本意的操作

  • 攻击者角度
    攻击者欺骗浏览器，让其以受害者的名义执行自己想要的操作
    利用用户已登录的身份，在用户毫不知情的情况下，以用户的名义完成非法操作

防止CSFR的方法

  • 检查Referer字段
  • 添加校验token

跟跨网站脚本（XSS）相比，XSS 利用的是用户对指定网站的信任，CSRF 利用的是网站对用户网页浏览器的信任

命令注入 Command Injection

命令注入流程：

1.判断是否调用系统命令

2.判断函数或函数的参数是否可控

3.判断是否拼接注入命令

  • 比如用&&或者&拼接两个命令
  • 比如用| 把第一个函数的值传给第二个函数

常用的符号

A&B： 简单的拼接，AB无制约关系

A&&B： A执行成功，然后才会执行B

A|B： A的输出，作为B的输入

A||B：A执行失败，然后才会执行B

如果过滤了敏感命令

比如：

过滤了whoami命令

windows可以使用"" who""am""i

Linux系统中还可以使用‘隔开 who'am'i

如果不显示输出结果怎么办

可以使用延迟注入，远程请求观察响应时间来判断是否有命令注入漏洞

延迟注入：

windows ping 127.0.0.1 -n 5 >nul

linux sleep 5 延迟5s

远程请求：
windows：ping，telnet等

Linux：wget，curl等

http简述

http是个啥

HTTP（Hypertext Transfer Protocol）中文名：超文本传输协议
Web使用http作为规范，完成客户端到服务器等一系列运作流程
HTTP协议的职责：生成针对目标Web服务器的HTTP请求报文

告知服务器意图的HTTP方法

1.GET：获取资源，从指定的资源请求数据（用来请求访问已被URI识别的资源，请求可被缓存）
2.POST：向指定的资源提交要被处理的数据（用来传输实体的主体，请求不会被缓存）
3.PUT：传输文件保存到URI指定位置
4.HEAD：获得报文头部（用于确认URI的有效性及资源更新的日期时间等）
5.DELETE：请求URI删除指定文件（与PUT相反）
6.OPTIONS：用来查询针对请求URI指定的资源支持的方法

http报文

报文是http协议中的基本单位，是用于http协议交互的信息

请求报文/响应报文的结构
|报文首部:http首部字段+请求行/状态行|
|:—–:|
|空行（CR+LF）|
|报文主体|
-请求行：包含用于请求的方法，请求URI和HTTP版本
-状态行：包含表明响应结果的状态码，原因短语和http短语
-首部字段：包含表示请求和响应的各种条件和属性的各类首部
首部字段结构
首部字段包括：请求/响应首部字段，通用首部字段，实体首部字段
首部字段名：字段值

通用首部字段
通用首部字段是指，请求报文和响应报文双方都会使用的首部
cache-control：能够控制缓存
缓存请求指令有：no-cache（强制向源服务器再次验证），no-store（不缓存请求或响应的任何内容），max-age=[秒]，no-transform（代理不可更改媒体类型），only-if-cached
缓存响应的指令：public，private，no-cache，no-store，no-transform，must-revalidate（可缓存但必须再向源服务器进行确认），max-age=[秒]

Connection：

  • 控制代理不再转发的首部字段
    connection：不再转发的首部字段名
  • 管理持久连接
    connection：close/keep-alive

Date表明创建http报文的日期和时间
Date：Tue, 05 Mar 2018 11:27:32 GMT
Transfer-Encoding:chunked(分块传输编码)

请求首部字段
请求首部字段是从客户端往服务端发送请求报文中所使用的字段，用于补充请求的附加信息，客户端信息，对响应内容相关的优先级等内容
Accept：通知能够处理的媒体类型
（text/html image/gif video/mpeg）
Accept-Charset：通知服务器用户代理支持的字符集
Accept-Ecoding：通知服务器用户代理支持的内容编码
Accept-Language：自然语言
Authorization：告知服务器用户代理的认证信息
Host：告知服务器请求资源所处的互联网主机名和端口号（必须包含的首部字段）
If-xxx:条件请求。服务器接收到附带条件的请求后，只有判断为真，才能执行请求
Referer：告知服务器请求的原始资源的URI
User-Agent：将创建请求的浏览器和用户代理名称等信息传达给服务器

响应首部字段
响应首部字段是由服务器向客户端返回响应报文中所使用的字段，用于补充响应的附加信息，服务器信息，以及对客户端的附加要求等信息
Accept-Ranges：用来告知客户端是否能处理范围请求，值有两种：bytes，none
Age：告知客户端，源服务器在多久前创建了响应，单位为秒
Location：可以将响应接受方引导至某个与请求URI位置不同的资源（常用于更换域名的时候）
Server:服务器用来处理请求的软件信息(与User-Agent请求相对应)

实体首部字段
实体首部字段是包含在请求报文和响应报文中的实体部分所使用的首部，用于补充内容的更新时间等与实体相关的信息
Allow：通知客户端能够支持Request-URI指定资源的所有HTTP方法
Content-Encoding：告知客户端服务器对实体的主体部分选用的内容编码方式
Content-language：资源所用的自然语言
Content-Length：实体主体部分的大小，单位为字节
Content-Location：给出与报文主体相对应的URI
Content-Type：发送给接收者的实体正文的媒体类型
Expires：将资源失效的日期告知客户端
Last-Modified：资源最终被修改的时间

状态码告知从服务端返回的请求结果

类别 原因短语
1xx Information（信息型状态码） 接受的请求正在处理
2xx Success（成功状态码） 请求正常处理完毕
3xx Redirecttion（重定向状态码） 需要进行附加操作以完成请求
4xx Client error（客户端错误状态码） 服务器无法处理请求
5xx Server error（服务器错误状态码） 服务器处理请求出错

常见的状态码

1.200 OK：客户端发来的请求在服务器端被正常处理了
2.204 NO Content：请求处理成功，但没有资源可以返回
3.206 Partial Content：表示客户端进行了范围请求，服务器成功执行了这部分的请求
4.301 Moved Permanently：永久性重新定向，表示请求的资源已被分配了新的URI
5.302 Found：临时性重新定向，资源的URI临时定位到其他位置了
6.303 See Other：表示请求对应的资源存在另一个URI，应使用GET方法定向获取请求的资源
7.304 Not Modified：资源已找到，但未符合条件要求
8.400 Bad Request：表示请求报文中存在语法错误
9.401 Unauthorized：未经授权，发送的请求需要通过认证
10.403：Forbidden：表示对请求资源的访问被服务器拒绝了
11.404 Not Found：服务器上没有请求的资源（也可以用在服务器端拒绝请求且不想说明理由）
12.500 Internal Server Error：服务器端在执行请求时发生了错误
13.503 Service Unavailable：服务器暂时处于超负载或正在进行停机维护

代理，网关，隧道

代理：接受客户端的请求后转发给其他服务器
使用代理的理由

  • 利用缓存技术减少网络带宽的流量
  • 组织内部针对特定网站的访问控制
  • 获取访问日志

缓存代理：预先将资源的缓存保存到代理服务器上，当再次接受到相同资源请求直接将缓存的资源作为响应返回
透明代理：不对报文做任何加工，反之为非透明代理

网关：工作机制与代理类似
使用网关的理由：

  • 能使通信线路上的服务器提供非HTTP协议服务
  • 提高通信的安全性，可以在客户端与网关的通信线路上加密以确保连接安全

隧道：在相隔甚远的客户端和服务器两者之间的中转
目的是确保客户端能与服务器进行安全的通信，隧道是透明的
